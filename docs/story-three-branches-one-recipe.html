Branch 1 997b85e

Bounded Parallelism

parallel: N on foreach steps ‚Äî concurrent execution with controllable limits.

Before

steps: - foreach: items agent: reviewer # runs one at a time # 10 items = 10√ó latency

After

steps: - foreach: items agent: reviewer parallel: 5 # 5 concurrent workers # bounded, not unbounded

Why bounded?

Unbounded parallelism hits rate limits and overwhelms providers. parallel: N lets recipe authors tune concurrency to their provider quotas.

Provider Preferences

Fallback chains and per-step model selection ‚Äî the right model for each task.

Per-step selection

Heavy reasoning steps use capable models. Boilerplate steps use fast, cheap models. Cost scales with task complexity, not recipe length.

üõ°Ô∏è

Fallback chains

If the primary provider is down or rate-limited, execution continues on the fallback ‚Äî no recipe failure.

üåê

Glob patterns

claude-haiku-* matches the latest haiku variant. Recipes stay current without hardcoding model versions.

Convergence Loops

While-conditions and convergence checks ‚Äî recipes that iterate toward quality.

Meta-learning pattern

Agents evaluate their own output and decide whether to iterate. The recipe defines the convergence criteria; the agent finds the path.

Bounded iteration

max_iterations prevents infinite loops. Convergence is a goal, not a guarantee ‚Äî the executor always terminates.
