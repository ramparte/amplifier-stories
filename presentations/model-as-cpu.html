The Model Is
the New CPU

A thesis on the disappearance of software.
The demo is running today.

Amplifier Team · March 2026

The Familiar

What does a CPU do?

Strip it down to first principles.

⬇️
Receives Input

A keystroke. A click. An electrical signal. Raw data arrives.

⚡
Decides

Routes, computes, transforms. The instruction set determines what happens next.

⬆️
Executes

Produces output. Changes state. The world is different after.

Input → Decision → Execution. That's it. Everything else is abstraction.

The Parallel

Now ask the same question
about a model.

💬
Receives Input

A prompt. A voice command. An uploaded file. Raw intent arrives.

🧠
Decides

Reasons, plans, chooses. The model determines what to generate.

✨
Executes

Produces an application. A response. An action. The world is different after.

Same pattern. Same function. Different substrate.

No APIs. No predefined logic. No explicit software layer in between.

We keep framing AI as a tool
that helps humans write software.

The frame is wrong.

The model doesn't assist with software.
It replaces software.

The Proof

A mini PC. A living room. A blank canvas.

How It Works

The system learns you.

It doesn't just generate once. It remembers. It adapts. It builds context over time.

🎨
Preferences

Remembers your preferred colors, layouts, interaction patterns. Every generated app reflects what you like, not factory defaults.

💾
Persistence

Data stored in JSON. Persists across sessions. Your notes, your data, your context — always available.

🛠️
Developer Panel

Want to tweak the generated app? A developer panel lets you inspect, customize, and modify what was built.

♻️
Lifecycle

Apps aren't permanent installations. Use them, delete them, restore from trash. Software becomes ephemeral.

The Everyday Test

Renewing a passport.

Today

Find the government website. Navigate menus. Download a form. Fill it out. Upload a photo. Pay online. Track status in a different system. Print confirmations.

Multiple steps. Multiple software systems. Friction at every turn.

With a Model-CPU

"Renew my passport." The AI knows who you are. Has your permissions. Understands the goal. It executes.

No forms. No software. No steps.

If the model knows the user, has the right permissions, and understands the goal — it just executes. The software layer was never the point. It was just the interface we needed because machines couldn't understand intent.

The Analogy

You manage a team of 10.

You
Give a directive
→
They
Execute

You don't need to understand their internal processes. You don't micromanage the method. You state the goal. They deliver the outcome.

The model is your team.

The Consequence

When software disappears,
what remains?

📦
Software Gets Commoditized

If any model can generate any application on demand, software loses its strategic value. The competitive moat of "we built better software" erodes.

🔩
Hardware Is the Last Moat

The physical device — what you hold, what processes, what connects — becomes the last defensible asset. The thing you can't generate from a prompt.

🍎
Apple Already Knows

Apple's strategy makes more sense in this frame: invest in hardware, own the device, let the software layer become increasingly fluid and model-driven.

If this thesis holds, the companies that win aren't the ones writing the best code. They're the ones building the best silicon.

The Bottleneck

One constraint
holds this back.

Token speed.

Today
Thousands/sec
Threshold
Millions/sec
Thousands of tokens/sec

The model is an assistant. It helps. It suggests. You wait. The latency gap between "intent" and "application" is perceptible. Software still mediates.

Millions of tokens/sec

The model is a CPU. Intent becomes application instantly. The latency disappears. There is no gap to fill with traditional software.

As models accelerate toward millions of tokens per second, the functional distinction between "AI assistant" and "CPU" collapses.

Are we building the
last generation
of traditional software?
