“Could two teams want different behavior? Then it's a module, not the kernel.”

Landing Page · Developer Journey

“The Experimental Layer”
Understand it. Use it. Make it yours.

The page's second half is a developer funnel — three stages from comprehension to ownership, backed by fully-built documentation pages.

Stage 1

Understand it

Three concept cards link to full documentation routes: The Core (sessions, initialize, execute), The Modules (5 protocols, mounting, resolution), The Foundation (bundles, composition, prepare/create). Each with code blocks, ASCII diagrams, and TOC navigation.

Stage 2

Use it

Three experiment cards linking to real source on GitHub: inject a hook (Hooks API), swap a provider (Provider contract), replace the loop (Orchestrator contract). Hands-on from day one.

Stage 3

Make it yours

Two paths: build your app with AmplifierSession (Python), or automate with recipes (YAML with staged approval gates). From user to contributor.

Final CTA: “Try a reference implementation. The Amplifier CLI.” — install command + View on GitHub
