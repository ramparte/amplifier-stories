The Capabilities Gap

It’s not about the cost.
It’s what the cost unlocks.

📏

Million-Token Contexts

1 million token context windows as a hard requirement. Newton’s team evaluated tools with 256K windows — and ruled them out as insufficient for their workflows.

🚫

“Stop Writing Code Manually”

A team-wide cultural mandate that only makes sense with unlimited access. Not a suggestion — a workflow directive. One that can’t function under quota pressure.

🏗️

Architecture-Level Dependence

Workflows and architectures designed around AI availability. Not AI-augmented development — AI-dependent development. The model is a load-bearing wall.

The Infrastructure Layer
429
Too Many Requests

Even with unlimited subscriptions, Newton’s team hits rate-limit errors. Not quota exhaustion — infrastructure saturation. The API says slow down even when the contract says unlimited.

This forces routing strategies across providers — Anthropic, OpenAI, and increasingly, local models — not as a cost optimization, but as an availability strategy.

“There’s a different type of scarcity now — GPU scarcity.”
Satya Nadella, on structural compute constraints even for well-funded teams
The Overflow

Local models are now viable

Not a replacement. An overflow valve. And the quality trajectory has been dramatic.

For Newton’s team

A routing target when cloud APIs return 429s. Run locally, route back to cloud when capacity frees up. Seamless degradation.

For Faraday

A potential escape from the quota clock. But running capable local models requires hardware that creates its own access gap.

The AI productivity revolution is real.

But it is not evenly distributed.

The developers building the most ambitious AI-native workflows are doing so on infrastructure that costs more per day than many developers earn in a week.

The Question

Who gets to pioneer the future of software development?

And who is watching from behind a quota wall?

This conversation is happening right now. Between friends.
